# Changelog

## 0.4.2 - Update Flow Reliability

- 新增 `normalize_version` 与 `compare_versions`，正确处理 `0.10.0 > 0.2.0` 等语义化版本比较。
- `linux-tool update` 在版本一致时明确提示当前已是最新版本，不再继续下载、备份或覆盖。
- 启动时静默检查更新：版本一致只写 INFO 日志，网络失败不阻断进入 TUI。
- fzf 的 Ctrl+U 改为由主 Bash 程序接收动作后执行更新；更新成功后会重新执行 `linux-tool`。
- 修复 git 工作区更新后误删 `tool/` 的风险，安装时会把内置工具补入本地工具目录但不覆盖已有工具。
- 优化启动 Logo 尺寸和 TUI 顶部提示，减少小窗口占用。

## 0.4.1 - TUI Header Layout

- Logo 改为 `linux` 与 `tool` 上下两段显示。
- fzf 顶部固定区改为 Logo、信息、分隔线、`选项 | 说明` 的层级。
- 工具列表显示改为 `选项 | 说明` 样式，左侧更像菜单，右侧继续显示预览说明。

## 0.4.0 - Fast Raw Cloud TUI

- 云端发现改为 GitHub tree 页面索引 + raw 文件下载，不再使用 GitHub API 或 tools.txt。
- 启动时只缓存云端文件名，预览/安装时才下载对应 raw 脚本，减少 TUI 启动等待。
- 优化 fzf 顶部固定 Logo、项目链接、快捷键和响应式预览布局。
- 云端、本地、分类标签加入 ANSI 彩色显示。
- 安装和更新流程增加 spinner 与更清晰的步骤状态。
- Logo 改为彩色粗体填充风格，默认使用本地静态/缓存，避免启动时实时 npx 拖慢。

## 0.3.0 - Cloud Tool Catalog

- 工具中心本体与具体工具脚本解耦，安装脚本不再把 `tool/` 复制为本地已安装工具。
- 新增云端工具检测、云端预览、按需安装和本地卸载。
- fzf 顶部常驻 Logo、版本、路径和快捷键说明。
- 本地已安装工具与云端可安装工具合并展示，云端列表自动跳过已安装项。
- 更新逻辑兼容旧安装用户，保留本地已安装工具，不再从应用包同步 `tool/`。

## 0.2.0 - Linux Tool Center

- 重构为本地 Linux 终端应用。
- 新增 `bin/linux-tool` 主程序。
- 新增 `lib/` 公共模块：工具加载、fzf UI、预览、Logo、日志、安全确认、更新和版本。
- 工具统一从 `tool/*.sh` 自动发现。
- 安装路径切换到当前用户的 `~/.local` 目录，不再强制 root。
